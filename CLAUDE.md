## Project Overview

A Rust library that watches cert-manager Certificate resources in a Kubernetes cluster and automatically serves an Axum router over TLS with certificate rotation. Used as a dependency, not a binary.

## Commands

```bash
cargo build           # Build the library
cargo check           # Fast syntax/type check
cargo test            # Run tests (none exist yet)
cargo clippy          # Lint
cargo fmt             # Format code
cargo fmt --check     # Check formatting without modifying
cargo doc --open      # Build and view docs
```

## Architecture

Three async components run concurrently, joined in `serve_cert_manager_https()` (`src/lib.rs`):

1. **`CertController`** (`src/controller.rs`) — Kubernetes controller (kube-runtime) that watches `Certificate` CRDs in a namespace. On certificate events, fetches the associated Secret, builds a `rustls::ServerConfig`, and sends an `AcceptorWithExpiration` over a `tokio::sync::mpsc` channel.

2. **`AcceptorStream`** (`src/accept.rs`) — Listens on a TCP socket, polls the mpsc channel for new acceptors, upgrades TCP connections to TLS. Only replaces the current acceptor if the incoming certificate expires later (prevents downgrade). Manages in-flight TLS handshakes via `FuturesUnordered`.

3. **`ServerConnErrorStream`** (`src/server.rs`) — Consumes completed TLS streams from `AcceptorStream` and serves the caller-provided `axum::Router` via hyper-util (HTTP/1.1 + HTTP/2).

### Data flow

```
K8s cert-manager Secret → CertController → AcceptorWithExpiration (mpsc)
                                                      ↓
TCP listener → AcceptorStream (TLS upgrade) → ServerConnErrorStream → Axum Router
```

### CRD types

`src/crd.rs` contains cert-manager `Certificate` and related types generated by [Kopium](https://github.com/kube-rs/kopium). Regenerate with:
```bash
kopium -f <crd-yaml> > src/crd.rs
```

## Key Details

- **TLS crypto**: `rustls` + `aws-lc-rs` (FIPS-compatible path, post-quantum preferred). `aws-lc-rs` requires no extra system dependencies in the default (non-FIPS) build.
- **Kubernetes version**: Selected via Cargo features `v1_31`–`v1_35` (default: `v1_35`). Set the feature matching your cluster.
- **RBAC**: The deployed pod needs permission to `watch` `certificates.cert-manager.io` and `get`/`list` `secrets` in the target namespace.
